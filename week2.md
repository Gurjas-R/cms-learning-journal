## Content Insights
This module I learned the differencfe between static and dynamic websites. Static sites are made of idividual HTML files and have to be updated manually, whicb is time consuming and fustrating if major changes need to be made and have to be copied through each page and is challenging for non-technical users to navigate and work on. Dynamic websites generate pages using server-side code and a database, meaning content can be updated easily and allows multiple developers to manage a website. CMS softwares like Joomla provide this for creating, updating and managing content without writing or touching a single piece code, which is ideal for non-technical users. This is helpful as in a scenario for me I had to change the navigation bar to fix it to get it work on Joomla but to do it with html I would've had to change the href to each page by hand if it was static as the file names changed, However on cms you can make it a global piece of code that changes it all files and still works even if the pages names are changed as it will automatically update which was the best improvement I noticed for my use case. I learned further that plugins and templates can allow users to create easy CSS styles or features that would normally definitely require code and not just simple code but complicated code.
