# Week 6 – Project Management & Team Workflow

## Learning Activities & Resources
This week I learned about project management and how teams organise and manage web development projects. Then as a group we set up GitHub for version control and used a trello board to organise tasks and track progress.


## Estimated Hours
2 hours

## Content Insights
Large projects should be broken down into smaller tasks so that progress can be tracked more easily and each team member knows what to work on. Systems like GitHub are used to track changes, manage different versions of a project, and allow multiple people to work on the same project without overwriting each other’s work. Another important concept Stand-up meetings, where each team member explains what they did, what they are working on next, and if they have any problems. This helps teams communicate and avoid delays.

## Career / Employability / Learning Insights
This week helped me understand how software development teams work in real jobs. Developers do not work alone on large projects, so communication and task management are important. Using GitHub is important because it allows teams to collaborate and keep track of changes. I also learned that project management tools help teams stay organised and complete projects on time. These are important skills for working in the IT industry because most development teams use version control and project management tools.
